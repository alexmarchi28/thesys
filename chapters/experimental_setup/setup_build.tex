\section{Compilation and Configuration of the Environment}

The adopted build sequence is reported in this section in reproducible form. The order is important because some projects require artifacts compiled from previous dependencies.

\subsection{OMNeT++ 6.2}

For OMNeT++, the official workflow based on \texttt{install.sh} and subsequent local compilation was followed \cite{omnet_download,omnet_versions}. A minimal example is reported below.

\begin{lstlisting}[caption={OMNeT++ 6.2 installation and build},label={lst:omnet_build}]
tar -xvf omnetpp-6.2.0-src-linux.tgz
cd omnetpp-6.2.0

# Installs dependencies and creates the Python virtualenv
# Also runs configuration, compilation, and environment import automatically
./install.sh
\end{lstlisting}

\subsection{SUMO 1.20 with updated dependencies}

The workflow used in the thesis was:
\begin{enumerate}
\item install the dependencies indicated by the current SUMO repository;
\item check out version 1.20;
\item compile from source with CMake.
\end{enumerate}

\begin{lstlisting}[caption={Build of SUMO 1.20 with dependencies from the official repository},label={lst:sumo_build}]
git clone https://github.com/eclipse-sumo/sumo.git
cd sumo

# Dependencies (official SUMO repository)
sudo apt-get install \$(cat build_config/build_req_deb.txt build_config/tools_req_deb.txt)

# Switch to version 1.20 (v1.20 is an arbitrary name for the created branch)
git checkout -b v1.20 v1\_20\_0

# Compilation
cmake -B build .
cmake --build build -j$(nproc)

\end{lstlisting}

\subsection{INET, Simu5G, Veins, Plexe, and Subprojects}

For the other components, the official documentation was followed, with particular reference to the instructions in \texttt{plexe\_5g} for the INET $\rightarrow$ Simu5G $\rightarrow$ Veins $\rightarrow$ Plexe chain \cite{plexe_5g_readme,plexe_building}.

\begin{lstlisting}[caption={Build sequence of the OMNeT++-based frameworks},label={lst:framework_build}]
# INET
git clone https://github.com/inet-framework/inet
cd inet/
git checkout -b inet_v4.5.4 v4.5.4
source setenv

make makefiles
make -j <ncores> MODE=release
make -j <ncores> MODE=debug

# Simu5G (branch indicato da plexe_5g)
git clone https://github.com/michele-segata/Simu5G/ simu5g
cd simu5g/
git checkout plexe_mec
source setenv

make makefiles
make -j <ncores> MODE=release
make -j <ncores> MODE=debug

# Veins (+ eventuale sotto-progetto veins_inet)
git clone https://github.com/sommer/veins/
cd veins/
git checkout -b veins_v5.3.1 veins-5.3.1
source setenv

./configure
make -j <ncores>

# veins_inet
cd subprojects/veins_inet/
./configure
make -j <ncores>

# Plexe + subprojects
git clone https://github.com/michele-segata/plexe
cd plexe/
git checkout -b plexe_v3.2 plexe-3.2
source setenv

./configure
make -j <ncores>

cd subprojects/plexe\_5g
./configure
make -j <ncores>

\end{lstlisting}
