\section{Validation of the Development Environment}

Initial validation was set up as a progressive \emph{smoke test}: first basic Plexe and Veins examples, then scenarios with heterogeneous interfaces, and finally verification of the subprojects.

The official tutorials provide configurations that are useful for a quick end-to-end pipeline check \cite{plexe_tutorial,plexe_example6}. The commands used are of the following type:

\begin{lstlisting}[caption={Execution of tutorial scenarios from the CLI (Cmdenv)},label={lst:smoke_commands}]
# Test Plexe / Veins (802.11p)
cd /path/to/plexe/examples/platooning
plexe_run -u Cmdenv -c Sinusoidal -r 2

# Test plexe_vlc
cd /path/to/plexe/subprojects/plexe_vlc/examples/plexe_vlc
./run -u Cmdenv -c Sinusoidal -r 0

# Test plexe_5g / Simu5G
cd /path/to/plexe/subprojects/plexe_5g/examples/platooning_5g
./run -u Cmdenv -c CV2XMerge -r 0
cd ../platooning_mec
./run -u Cmdenv -c Overtake -r 0

\end{lstlisting}

At this stage, the verification is not only that the simulation starts, but also includes:
\begin{itemize}
\item verification of intra-platoon communication by using the debugger available in OMNeT++;
\item correct TraCI handshake between OMNeT++ and SUMO;
\item generation of \texttt{.sca}/\texttt{.vec} files and logs consistent with the selected configuration;
\item plausible dynamic behavior of the platoon.
\end{itemize}
