\section{OMNeT++: Architecture and Experimental Workflow}
\label{env:omnet}

OMNeT++ is the discrete-event simulator at the core of the experimental environment adopted in this thesis. Its relevance in networking research stems from its modular architecture and support for hierarchical model composition, which make it possible to describe complex systems as collections of modules connected through messages and timed events \cite{varga2008overview}. In the V2X domain, this modularity is particularly useful because it allows the integration of communication stacks, applications, control logic, and interfaces toward external traffic simulators.

From an architectural point of view, OMNeT++ separates:
\begin{itemize}
\item the structural description of the system, namely module topology and connections, typically expressed through NED files;
\item the behavioral logic of the modules, implemented in C++;
\item the configuration of simulations and experimental parameters, managed through \texttt{omnetpp.ini} files.
\end{itemize}

This separation makes it possible to define scenario and configuration variants without modifying the source code. In the adopted workflow, OMNeT++ therefore plays the role of coordinator of the entire simulation: it initializes the modules, schedules events, collects statistics, and manages interactions with external frameworks such as Veins, PLEXE, and Simu5G \cite{segata2023multi-technology, nardini2020simu5g}.
