\section{Migration to 5G}
\label{sec:s5g_5g}

In the reference work, the cellular component of the multi-technology system is based on LTE C-V2X in a network-assisted configuration under network coverage \cite{segata2023multi-technology,3gpp-21914-1400}. In the present work, the goal was not to change the functional role of this third interface, but to update its model: the cellular channel had to continue behaving as an infrastructure-managed technology complementary to 802.11p and VLC, while being implemented with a more recent 5G stack.

Simu5G represents the natural evolution of SimuLTE within the OMNeT++ ecosystem and makes it possible to move to a 5G model without breaking continuity with the simulation environment already in use \cite{virdis2014simulte,nardini2020simu5g}.

From a design point of view, the work consisted of integrating into the multi-technology scenario a 5G component that was already available in PLEXE subprojects, using it as a basis instead of building a new cellular interface from scratch. This made it possible to preserve logic consistent with the rest of the environment: cooperative beacons continue to be treated as platooning traffic, and the cellular network continues to represent the third available channel alongside 802.11p and VLC.

The key point is that the migration to 5G does not modify the higher-level logic of \emph{SafeSwitch}. Followers continue to evaluate the reliability of the cellular component together with the other interfaces, and its contribution continues to be observed through metrics such as PDR, exactly as happens for the other channels. What changes is the underlying layer through which this interface is modeled: the effects of coverage, mobility across cells, and radio transients no longer depend on the previous LTE stack, but on the new model provided by Simu5G.
